# 存储段代理：对象存储与文件链路工作说明

## 你的角色

你负责项目的“文件上传与存储链路”。  
你的目标不是做业务逻辑，而是确保：

- 文件能安全上传
- 元数据可追踪
- 本地开发可模拟
- 生产对象存储可切换
- 服务端能稳定拿到文件地址与文件身份信息

---

## 一、你的边界

### 你负责

- 存储抽象接口 `StorageProvider`
- 本地 MinIO 适配
- 生产七牛云适配
- 文件对象命名规则
- 上传策略/签名生成
- 文件确认逻辑所需的数据结构
- 文件元数据规范
- 文件安全限制（大小、类型、去重）

### 你不负责

- AI 解析
- 标签提取
- 知识图谱
- 用户学习统计
- 图谱可视化

---

## 二、你的首月目标

在首月，你只需要交付一个**可靠、可替换、可并行开发**的文件链路。

最小成功标准：

1. 前端能拿到上传策略
2. 文件能上传到 MinIO 或七牛云
3. 服务端能通过 `file_id` 找到该文件
4. 文件有元数据记录
5. 可限制非法文件类型与超大文件
6. 可按 `sha256` 做简单去重

---

## 三、推荐设计

## 1. 存储抽象层

统一定义接口，禁止业务代码直接写死到某家云厂商 SDK。

建议接口：

```python
class StorageProvider:
    def create_upload_policy(self, *, user_id: str, filename: str, mime_type: str) -> dict: ...
    def build_object_key(self, *, user_id: str, content_type: str, ext: str) -> str: ...
    def confirm_uploaded_object(self, *, object_key: str) -> dict: ...
    def get_public_url(self, *, object_key: str) -> str: ...
    def delete_object(self, *, object_key: str) -> None: ...
```

## 2. 双环境策略

- 本地开发：MinIO
- 生产部署：七牛云

要求：

- 接口返回字段一致
- 前端无感知
- 服务端只依赖统一结构

## 3. 文件命名建议

对象键命名建议：

```text
/{env}/{user_id}/{content_type}/{yyyy}/{mm}/{uuid}.{ext}
```

示例：

```text
/prod/u_001/problem/2026/03/7f3c2e1a.png
/dev/u_001/note/2026/03/1ac9df20.jpg
```

要求：

- 不能用原始文件名作为对象键
- 必须保证唯一性
- 能从路径上看出环境、用户、内容类型

---

## 四、文件元数据规范

你必须保证服务端最终能拿到以下字段：

| 字段 | 说明 |
|---|---|
| file_id | 文件记录 ID |
| user_id | 归属用户 |
| object_key | 对象存储路径 |
| storage_provider | minio / qiniu |
| bucket_name | 存储空间 |
| file_url | 访问地址 |
| mime_type | 文件类型 |
| size_bytes | 文件大小 |
| sha256 | 文件哈希 |
| original_filename | 原始文件名 |
| source_type | upload / import |
| upload_status | pending / uploaded / confirmed / failed |
| created_at | 创建时间 |

---

## 五、推荐接口协作方式

为降低前后端耦合，推荐公共流程如下：

### 步骤 1：前端申请上传策略

前端调用：

```text
POST /v1/files/upload-policy
```

返回：

- object_key
- upload_token / policy
- upload_url
- file_id（预创建）
- 限制条件

### 步骤 2：前端直传对象存储

- 前端将文件直接上传到对象存储
- 减少后端文件中转压力

### 步骤 3：前端确认上传完成

前端调用：

```text
POST /v1/files/confirm
```

提交：

- file_id
- object_key
- sha256
- size_bytes
- mime_type

### 步骤 4：服务端校验并更新状态

你需要提供配套逻辑，支持服务端：

- 确认对象存在
- 更新 `upload_status = confirmed`
- 生成 `file_url`

## 六、文件安全规则

必须做以下限制：

1. 白名单类型：
   - jpg / jpeg / png / webp / pdf / txt
2. 单文件大小上限：
   - 图片：10MB
   - PDF：20MB
3. 拒绝可执行文件
4. 可根据 `sha256 + user_id` 做简单去重
5. 对异常 MIME 类型做校验，不只相信前端扩展名

你要输出给运维代理：

## 九、你的验收方式

你必须给出以下可见结果：

### 验收 1：本地上传成功

- 本地启动后
- 上传一张图片
- MinIO 控制台可看到对象
- 数据库可看到对应 file 记录

### 验收 2：切换存储提供商不改业务代码

- 将环境变量从 `STORAGE_PROVIDER=minio` 改为 `qiniu`
- 不改前端业务接口
- 上传流程仍可工作

### 验收 3：非法文件被拒绝

- 上传 `.exe` 或超大 PDF
- 系统返回清晰错误
- 不产生 confirmed 状态记录
