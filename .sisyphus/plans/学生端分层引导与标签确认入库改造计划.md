# 学生端分层引导与标签确认入库改造计划

## 1. 目标

按用户要求，把学生端改造成以下结构：

1. 登录页面单独一层；
2. 登录后进入主工作层；
3. 主工作层内并列展示：
   - 上传笔记
   - 询问问题
   - 笔记管理
   - 搜索
4. 询问问题支持图片输入；
5. 上传笔记与图片后，先让 AI 给出标签；
6. 标签反馈给用户并允许修改；
7. 用户确认后，才载入笔记记录。

## 2. 已确认现实

### 2.1 当前前端现实

当前 `apps/frontend/student/src/App.vue` 仍是单文件单页：

- 登录
- 上传并创建解析任务
- 最近任务
- 图谱概览
- 搜索
- AI solve
- 错题/笔记列表

没有独立登录页，也没有登录后的主导航分层。

### 2.2 当前后端现实

当前后端上传主线是：

1. `/v1/files/upload-policy`
2. `/v1/files/upload-local`
3. `/v1/files/confirm`
4. `/v1/ingestions`
5. worker 异步 `process_parse_job`

也就是说：

> 现在是“先创建内容实体，再异步分类”，并没有“AI 标签预览 -> 用户修改 -> 确认后再入库”的前置确认 API。

## 3. 必须新增的能力

为了满足用户要求，最小必要能力如下：

### 3.1 后端

需要新增一个“上传预分类预览”接口，例如：

- 输入：文件信息 / 文本内容 / 目标类型 / 问题图片
- 输出：
  - 推荐 `entity_type`
  - 推荐 `content_category`
  - 推荐 `subject`
  - 推荐标题/归一化文本
  - 推荐 `knowledge_candidates`

这个接口只做 AI 标签预览，不创建 note/problem 记录。

### 3.2 前端

学生端需要拆成两层：

1. **登录层**
2. **工作台层**

工作台层至少包含 4 个并列视图：

1. 上传笔记
2. 询问问题
3. 笔记管理
4. 搜索

并新增“标签确认卡片”：

1. 展示 AI 推荐标签；
2. 允许用户编辑标签、科目、内容类型；
3. 用户点击确认后，再执行入库链路。

## 4. 最小实现顺序

### 第一步：补后端预览接口

目标：先解决“确认前置”能力缺失。

最小方案：

1. 新增预览 schema；
2. 新增一个 preview route；
3. 直接复用现有 `classify_content()`；
4. 返回标签候选和推荐内容属性；
5. 不创建 DB 实体。

### 第二步：重构 student 前端结构

目标：把登录与工作台分层。

最小方案：

1. 登录成功前只显示登录层；
2. 登录成功后显示主工作台；
3. 主工作台使用 tabs / segmented nav 管理四个并列视图。

### 第三步：接入上传笔记标签确认流

目标：上传笔记/图片后先看 AI 标签，再确认入库。

最小方案：

1. 选择文件 / 输入文本；
2. 调 preview 接口获得标签建议；
3. UI 展示可编辑标签与类型；
4. 用户确认后才走文件确认 + ingestion。

### 第四步：接入提问问题图片流

目标：提问支持文本和图片。

最小方案：

1. 新增 question 视图；
2. 支持输入问题文本；
3. 支持可选图片上传；
4. 用现有 solve 能力生成答案；
5. 若有图片则先做同一套标签预览/类型建议。

### 第五步：QA 与文档同步

需要做：

1. student 前端 build；
2. 手动走登录 / 上传笔记 / 标签确认 / 搜索 / solve；
3. 如可行，补 Playwright；
4. 更新 docs。

## 5. 本轮验收标准

以下条件全部满足才算完成：

1. 登录页面与工作台层分离；
2. 工作台存在上传笔记 / 询问问题 / 笔记管理 / 搜索 四个并列入口；
3. 上传笔记与图片时，能先看到 AI 标签结果；
4. 用户可修改标签后再确认入库；
5. 确认后能在笔记管理中看到记录；
6. 询问问题支持图片输入；
7. 构建通过，手动 QA 通过。
