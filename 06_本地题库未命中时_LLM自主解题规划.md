# 本地题库未命中时：LLM 自主解题规划

## 一、为什么需要这个能力

这个项目的核心不是单纯“搜题”，而是“先利用自己的学习资料与题库，再由 AI 补位”。

所以标准流程必须是：

1. 先搜本地题库
2. 能命中就优先返回本地结果
3. 命不中再触发 LLM 自主解题
4. LLM 输出只能标记为“参考解析”

这样做有 3 个好处：

- 保护本地题库价值
- 提高答案一致性
- 降低 AI 幻觉风险

---

## 二、触发条件

只有当以下条件同时满足时，才触发 LLM fallback：

1. 已经完成题目文本标准化
2. 已经执行本地题库检索
3. 本地题库没有高置信命中
4. 用户允许使用 AI 参考解析
5. 当前题目不是明显非法或无法解析内容

建议“未命中”标准：

- 关键词检索无结果；或
- Top1 匹配分数低于阈值；或
- 仅命中弱相关题，不能作为标准答案来源

---

## 三、推荐总流程

```text
题目输入
  -> OCR/文本清洗
  -> 题目标准化
  -> 本地题库检索
      -> 命中：返回本地结果
      -> 未命中：进入 LLM 自主解题
              -> 第一次求解
              -> 第二次自检/校验
              -> 生成学生可读版解析
              -> 返回前端
              -> 保存到 AI 参考解析记录
```

---

## 四、分阶段实现建议

## 阶段 1：首月必须实现

只做最小闭环：

- OCR/文本清洗
- 本地全文检索
- 未命中时调用 LLM
- 输出答案、步骤、涉及知识点、置信度
- 保存记录

## 阶段 2：可选增强

- 加入向量检索
- 多候选答案比对
- 模型自反思
- 解题质量评分
- 自动沉淀到“待人工复核题库”

---

## 五、首月推荐输出格式

要求 LLM 返回结构化 JSON：

```json
{
  "subject": "math",
  "question_type": "calculation",
  "final_answer": "x=2 或 x=-3",
  "solution_steps": [
    "整理方程",
    "使用求根公式",
    "代入并计算"
  ],
  "knowledge_points": [
    "一元二次方程",
    "求根公式"
  ],
  "confidence": 0.78,
  "warnings": [
    "本题未在本地题库中命中",
    "以下为 AI 参考解析"
  ]
}
```

---

## 六、推荐的两阶段求解法

为了降低幻觉风险，建议不是“一次直接出答案”，而是两阶段：

## 第 1 阶段：Solver

任务：

- 读取题目
- 判断科目、题型
- 输出答案与步骤
- 标注知识点
- 给出自评置信度

## 第 2 阶段：Verifier

任务：

- 检查 Solver 是否遗漏条件
- 检查最终答案与过程是否一致
- 找可能的计算/逻辑错误
- 给出最终采纳建议

最终由服务端合并，返回“最终参考解析”。

---

## 七、服务端应做的保护

LLM 解题结果必须带这些保护：

1. **明确标注 AI 参考**
2. **不能覆盖本地题库标准答案**
3. **不能自动进入正式题库**
4. **必须保存生成记录与模型信息**
5. **低置信度内容可以提示用户谨慎参考**

---

## 八、数据记录建议

建议单独记录 AI 解题行为，便于后续复盘：

- problem_hash
- original_question_text
- normalized_question_text
- local_search_status
- local_top_score
- solver_model
- verifier_model
- final_answer
- solution_steps_json
- confidence
- warning_text
- created_at

---

## 九、推荐前端展示方式

当触发 LLM fallback 时，页面要显示：

### 模块 1：触发说明

> 未在本地题库中找到高置信匹配结果，已为你生成 AI 参考解析。

### 模块 2：最终答案

清晰显示结论。

### 模块 3：步骤解析

分步骤显示，方便学生理解。

### 模块 4：涉及知识点

帮助系统进一步建立题目与知识点的关系。

### 模块 5：风险提示

例如：

> 该解析由 AI 生成，仅供学习参考，请以教材/老师标准答案为准。

---

## 十、题库搜索建议（首月）

首月建议按以下顺序检索：

1. 题目原文全文检索
2. 规范化文本全文检索
3. 按标签过滤后的检索
4. 再决定是否 fallback

这样比一开始就做复杂向量召回更稳。

---

## 十一、质量控制建议

你至少要做这 4 个控制点：

1. **阈值控制**
   - 本地命中分数达到阈值时，不要走 AI fallback
2. **解析自检**
   - 让 verifier 检查 solver 输出
3. **结果落库**
   - 所有 AI 解析都留痕
4. **人工抽检**
   - QA 从常见题型抽样测试 AI 解析质量

---

## 十二、你最终要实现的目标

> 当学生上传一道系统没有搜到的题时，系统不是“无结果”，而是能给出一份带风险提示、带知识点、带步骤、可回看、可复核的 AI 参考解析。
