# 执行留痕规范

## 1. 目标

本规范用于保证：后续每一次规划、设计、实现、验证、部署，都能在 `D:\vibe_coding\note\docs` 中找到依据和历史。

## 2. 文档分层

后续文档按以下类型维护：

1. **启动/计划类**
   - 工作启动记录
   - 总执行计划
   - 阶段拆解计划

2. **实现记录类**
   - 阶段实现记录
   - 关键模块实现记录
   - 数据库/接口/前端结构调整记录

3. **验证与测试类**
   - 本地验证记录
   - 远程部署验证记录
   - 冒烟测试记录
   - 回归测试记录

4. **问题与决策类**
   - 关键设计决策记录
   - 已知问题清单
   - 风险与绕行说明

## 3. 每次工作必须记录的内容

每次重要工作后，至少记录：

1. 本次目标；
2. 已完成内容；
3. 修改/新增的文件；
4. 验证方式；
5. 当前风险；
6. 下一步。

建议使用：

- `01_工作启动记录_比赛级完整成品.md`

- `21_远程部署与验证记录.md`

## 5. 特殊规则

1. 若实现过程中发现需求与旧文档冲突，必须先写“决策记录”再改代码；
2. 若验证失败，必须写明失败现象、定位过程、临时结论；
3. 若远程服务器不可登录，也必须形成验证结论文档，不得口头略过。
