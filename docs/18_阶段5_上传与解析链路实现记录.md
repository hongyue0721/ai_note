# 阶段 5：上传与解析链路实现记录

## 1. 本次目标

把学生端业务主链路的第一段真实落地：

1. 文件记录
2. ingestion 创建
3. parse job 查询

## 2. 本次已完成

### 2.1 新增模型

已新增：

- `notes`
- `problems`

用于支撑 ingestion 入口真正生成内容实体。

### 2.2 新增 schema

已新增：

- 文件上传/确认 schema
- ingestion schema
- parse job 返回 schema

### 2.3 新增接口

已新增：

1. `POST /v1/files/upload-policy`
2. `POST /v1/files/confirm`
3. `POST /v1/ingestions`
4. `GET /v1/parse-jobs/{job_id}`
5. `POST /v1/parse-jobs/{job_id}/retry`

### 2.4 远端真实验证通过

已在远端完成完整调用链路：

1. 用户登录；
2. 申请上传策略；
3. 确认上传；
4. 创建 ingestion；
5. 查询 parse job。

### 2.5 实际验证结果

已确认：

- 文件记录能落库；
- 文件确认后状态变为 `confirmed`；
- ingestion 会创建 `problem` 实体；
- ingestion 会创建 `parse_job`；
- `parse_job` 查询接口返回成功；
- 当前 parse job 初始状态为 `pending`。

## 3. 当前结论

学生端真实业务链路已经从“纯鉴权”推进到：

- 用户 -> 文件记录 -> 内容采集 -> parse job 创建

这一段已可用。
