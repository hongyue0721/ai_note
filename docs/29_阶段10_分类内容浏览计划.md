# 阶段 10：分类内容浏览计划

## 1. 目标

把当前“能上传并分类”的链路，继续推进成“上传后能浏览、能查看”的可用体验。

## 2. 本阶段最小交付

### 后端

1. 错题列表接口
2. 错题详情接口
3. 笔记列表接口
4. 笔记详情接口

要求：

- 返回分类信息
- 返回解析状态
- 返回基础文本内容

### 前端

1. 学生端增加最近错题/笔记列表区
2. 展示分类标签
3. 选中后展示详情卡片
4. 上传后能即时刷新内容列表

## 3. 当前策略

比赛级阶段先做：

- 列表 + 详情的组合式展示
- 不先做完整路由页跳转
- 先让用户看到“上传后的内容已经进入系统并能查看”

## 4. 验收标准

1. 上传后的错题/笔记能出现在列表中；
2. 列表项可展示分类和状态；
3. 可查看基础详情；
4. 上传-分类-查看这条链路可演示。
