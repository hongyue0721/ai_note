# 未完成任务优先级总表

## 说明

本清单用于指导下一阶段构建顺序。

排序原则：

1. 先补“没有它就不算完整产品”的核心能力；
2. 再补“强影响比赛展示效果”的能力；
3. 最后补“稳定性、部署、自动化”能力。

---

## P0：必须优先完成（核心能力缺口）

### 1. 真实 AI 分类 agent 接入

- 增加 OpenAI 配置项
- 增加 OpenAI client
- 增加分类 prompt
- 增加分类 service
- 将 worker 从 mock 分类改为真实 AI 分类
- 低置信度继续进入 review 队列

### 2. `/v1/solve` AI 解题接口

- 实现 solve prompt
- 实现 solve service
- 实现 `/v1/solve`
- 可选补 `solve_history`
- 返回结构化答案、步骤、知识点、置信度、风险提示

### 3. 搜索能力（后端 + 前端）

- 实现 `/v1/search`
- 学生端搜索页
- 搜索结果支持 problem / note / document 分类展示

---

## P1：强影响比赛展示效果（成品感缺口）

### 4. 图谱能力（后端 + 前端）

- 实现 `/v1/graph/overview`
- 实现 `/v1/graph/weak-tags`
- 学生端图谱页
- 至少支持弱点标签与知识点关系可视化

### 5. 学生端真正文件选择器上传

- 增加本地文件选择器
- 支持图片/文档选择
- 增加大小/格式校验
- 增加上传中状态与错误提示

### 6. 问题 / 笔记详情独立页面

- 详情从当前卡片展示升级为独立视图
- 展示分类、状态、知识点、解析结果
- 为后续 AI 参考解析留位置

### 7. 后台解析任务页

- 增加 parse job 列表接口
- 后台增加任务页
- 支持按 pending / failed / success 筛选

---

## P2：完整产品闭环（功能收口）

### 8. 问题 / 笔记更新接口

- `PATCH /v1/problems/{id}`
- `PATCH /v1/notes/{id}`

### 9. AI fallback 展示闭环

- 本地未命中提示
- AI 参考解析展示模块
- 风险提示
- 置信度展示

### 10. 文档预览与内容细化展示

- PDF / 文档资料基础预览
- 文档类 note/detail 展示优化

---

## P3：部署与稳定性（可持续运行能力）

### 11. Docker Compose 全链路联调

- postgres
- backend
- worker
- student frontend
- admin frontend

### 12. 远端长期驻留部署

- systemd 或等效方式
- 反向代理（nginx/caddy）
- 静态前端托管

### 13. 自动化测试最小集

- API smoke tests
- E2E 基础流程测试
- 上传/登录/审核/任务流回归测试

## 推荐执行顺序

### 第一波（最值当）

1. 真实 AI 分类 agent
2. `/v1/solve`
3. 搜索 API + 搜索页

### 第二波（展示力提升最大）

4. 图谱 API + 图谱页
5. 文件选择器上传
6. 详情独立页面

### 第三波（成品闭环）

7. 后台解析任务页
8. 更新接口
9. AI fallback 展示

### 第四波（上线能力）

10. Docker Compose 全链路
11. 远端长期部署
12. 自动化测试

## 当前最推荐的下一步

因为这是当前从“能演示流程”迈向“真正智能产品”的最大分水岭。
