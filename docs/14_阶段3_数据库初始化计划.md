# 阶段 3：数据库初始化计划

## 1. 目标

让当前后端具备最小可运行的数据基础：

1. 能自动建基础表；
2. 能初始化默认管理员；
3. 为后续普通用户注册/登录、管理员登录提供真实数据支撑。

## 2. 本阶段最小交付

1. 建表脚本或启动时初始化能力；
2. 默认管理员 seed；
3. 启动时可选执行初始化；
4. 验证数据库连接与建表成功。

## 3. 当前策略

因为项目还在比赛级快速交付阶段，先采用：

- **启动期自动初始化基础表**
- **启动期自动检查/插入默认管理员**

后续再补 Alembic 正式迁移链路。

## 4. 默认管理员要求

默认需要具备：

- 用户名
- 密码
- 展示名

这些值应尽量从环境变量读取，避免硬编码死在代码里。

## 5. 验收标准

1. 远端启动后基础表存在；
2. 默认管理员可被创建；
3. 管理员登录接口具备真实数据支撑。
