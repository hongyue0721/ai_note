# 阶段 9：数据库轻量补列记录

## 1. 问题现象

在远端验证“图片/文档上传与分类”时，接口报错：

- `files.file_kind` 不存在
- `files.content_category` 不存在

## 2. 根因

当前后端使用的是：

- `Base.metadata.create_all()`

它可以创建新表，但**不会自动给已有表补新列**。

因此虽然代码模型已经增加了：

- `file_kind`
- `content_category`
- `category`

但远端旧数据库表结构还没有同步升级。

## 3. 当前处理策略

为了保持比赛级推进速度，当前不先强制补完整 Alembic 迁移，而是先在启动初始化中加入轻量 SQL patch：

- `ALTER TABLE ... ADD COLUMN IF NOT EXISTS ...`

## 4. 影响

这样可以确保：

1. 已存在的远端表结构能被自动补齐；
2. 当前图片/文档上传分类链路继续推进；
3. 不阻塞后续前端与业务验证。
