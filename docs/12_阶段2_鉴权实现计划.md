# 阶段 2：鉴权实现计划

## 1. 阶段目标

把“普通用户体系 + 独立管理员体系”的设计，推进为可实际调用的接口能力。

## 2. 本阶段要完成的最小交付

### 普通用户侧

1. 用户注册接口
2. 用户登录接口
3. `GET /v1/me`

### 管理员侧

1. 管理员登录接口
2. 管理员自检接口（用于后台判断登录态）

## 3. 设计要求

1. 普通用户与管理员使用不同表；
2. 普通用户与管理员 token 使用不同 secret；
3. 两类 token 必须带不同 scope；
4. 接口返回结构必须统一；
5. 为后续数据库正式接入前，允许先提供 seed/初始化能力或占位实现，但结构必须能平滑过渡。

## 4. 本阶段不做

1. 完整权限矩阵；
2. 找回密码；
3. 邮箱验证；
4. 复杂会话管理。

## 5. 验收标准

1. 普通用户注册/登录接口存在；
2. 管理员登录接口存在；
3. `me` 类接口能区分普通用户与管理员；
4. 代码可在远端完成导入级校验。
