## 4. 当前已知技术路线

### 后端
- FastAPI
- SQLAlchemy
- PostgreSQL
- Worker 轮询 `parse_jobs`

### 存储
- 本地：MinIO
- 生产/演示：Qiniu 或兼容对象存储

### 前端
- 学生端：沿现有前端规划，优先 Web/H5
- 后台端：单独前端项目
- 可视化：ECharts

### 监控
- 比赛级基础监控，内建在后台中
- 不引入 Prometheus + Grafana 重型体系

## 5. 关键约束

1. 每完成一个阶段必须写文档；
2. 不扩展到商业级产品范围；
3. 不把管理员账号体系和学生账号体系合并；
4. 后台不做完整内容运营平台；
5. 测试采用“先功能后测试”的顺序，但最终必须补齐最小可验证测试。
