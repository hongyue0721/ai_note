# 接口文档（按当前真实代码生成）

## 1. 说明

本文件基于当前真实代码生成，而不是基于早期计划文档推断。

统一约定：

- 后端基础前缀：`/v1`
- 健康检查：`/healthz`
- 响应外层统一为：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

- 用户侧鉴权：`Authorization: Bearer <user_token>`
- 管理员侧鉴权：`Authorization: Bearer <admin_token>`

## 2. Health / Meta

### `GET /healthz`

用途：服务健康检查。

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": { "status": "ok" }
}
```

### `GET /v1/meta/app`

用途：读取应用元信息。

响应字段：

- `app_name`
- `env`
- `mvp_mode`

## 3. 用户鉴权 / 管理员鉴权

### `POST /v1/auth/register`

请求体：

```json
{
  "username": "string(3-64)",
  "email": "string|null",
  "password": "string(6-128)"
}
```

响应：`UserProfile`

- `id`
- `username`
- `email`
- `status`

### `POST /v1/auth/login`

请求体：

```json
{
  "username": "string",
  "password": "string"
}
```

响应：`UserLoginData`

- `access_token`
- `token_type`（固定 `bearer`）
- `user`

### `GET /v1/me`

鉴权：用户 token。

响应：`UserProfile`

### `POST /v1/admin/auth/login`

请求体：

```json
{
  "username": "string",
  "password": "string"
}
```

响应：`AdminLoginData`

- `access_token`
- `token_type`
- `admin`

### `GET /v1/admin/me`

鉴权：管理员 token。

响应：`AdminProfile`

- `id`
- `username`
- `display_name`
- `status`

## 4. 文件上传

### `POST /v1/files/upload-policy`

鉴权：用户 token。

请求体：

```json
{
  "filename": "string",
  "mime_type": "image/png|image/jpeg|application/pdf|text/plain|application/msword|application/vnd.openxmlformats-officedocument.wordprocessingml.document",
  "content_type": "problem|note|document",
  "file_kind": "image|document"
}
```

响应：

- `file_id`
- `object_key`
- `upload_url`
- `upload_token`
- `max_size_bytes`
- `accepted_as`

### `POST /v1/files/upload-local`

鉴权：用户 token。

表单字段：

- `file_id`
- `object_key`
- `upload_file`（multipart file）

响应：

- `file_id`
- `object_key`
- `bytes_written`

### `POST /v1/files/confirm`

鉴权：用户 token。

请求体：

```json
{
  "file_id": "uuid",
  "object_key": "string",
  "size_bytes": 123,
  "mime_type": "string",
  "sha256": "string"
}
```

响应：

- `file_id`
- `file_url`
- `upload_status`
- `file_kind`
- `content_category`

## 5. Ingestion / Parse Jobs

### `POST /v1/ingestions`

鉴权：用户 token。

请求体：

```json
{
  "entity_type": "problem|note",
  "file_id": "uuid|null",
  "text_content": "string|null",
  "subject": "string",
  "source_type": "string",
  "content_category": "problem|note|document",
  "auto_parse": true
}
```

约束：`file_id` 与 `text_content` 不能同时为空。

响应：

- `entity_type`
- `entity_id`
- `parse_job_id`
- `parse_status`
- `content_category`

### `GET /v1/parse-jobs/{job_id}`

鉴权：用户 token。

响应：

- `id`
- `status`
- `entity_type`
- `entity_id`
- `content_category`
- `result`
- `error_message`
- `attempt_count`
- `created_at`
- `updated_at`

### `GET /v1/admin/parse-jobs`

鉴权：管理员 token。

查询参数：

- `status`（可选）

响应：`ParseJobListItem[]`

### `POST /v1/parse-jobs/{job_id}/retry`

鉴权：管理员 token。

响应：

- `id`
- `status`
- `attempt_count`

## 7. Review / Dashboard / Monitor

### `GET /v1/review/tasks`

鉴权：管理员 token。

查询参数：

- `task_type`（可选）
- `status`（可选）
- `entity_type`（可选）

响应：`ReviewTaskItem[]`

### `POST /v1/review/tasks/{task_id}/decision`

鉴权：管理员 token。

请求体：

```json
{
  "action": "approve|reject|replace",
  "edited_tags": []
}
```

响应：

- `id`
- `status`

### `GET /v1/dashboard`

鉴权：用户 token。

响应：

- `today_problem_count`
- `today_note_count`
- `pending_review_count`
- `pending_parse_job_count`
- `failed_parse_job_count`

### `GET /v1/admin/monitor/overview`

鉴权：管理员 token。

响应：

- `service_status`
- `parse_job_total`
- `parse_job_pending`
- `parse_job_failed`
- `review_task_pending`
- `latest_error_messages`

## 8. Search / Graph / Solve

### `GET /v1/search`

鉴权：用户 token。

查询参数：

- `q`（必填，最小长度 1）

响应：`SearchItem[]`

### `GET /v1/graph/overview`

鉴权：用户 token。

响应：

- `nodes[]`
  - `name`
  - `weight`
- `total_problems`
- `total_notes`

### `GET /v1/graph/weak-tags`

鉴权：用户 token。

响应：

- `name`
- `score`

### `POST /v1/solve`

鉴权：用户 token。

请求体：

```json
{
  "question_text": "string",
  "subject": "string|null",
  "allow_ai_fallback": true
}
```

响应：

- `subject`
- `question_type`
- `final_answer`
- `solution_steps[]`
- `knowledge_points[]`
- `confidence`
- `warnings[]`
- `model`

## 9. 当前接口层结论

1. 用户鉴权与管理员鉴权接口都已具备；
2. 用户主流程接口已经完整覆盖：登录、上传、ingestion、parse、内容列表/详情、搜索、图谱、solve；
3. 管理员主流程接口已经完整覆盖：登录、监控、审核、parse jobs；
4. 当前更大的问题不再是“接口缺失”，而是前端产品化深度与浏览器级 E2E 仍未做满。
