# 真实 AI 失败路径回归验证记录

## 1. 本次目标

在真实 AI 主链路已经跑通的前提下，继续补齐“失败时系统会如何报错”的实际证据，避免后续调试再次遇到模糊 500。

## 2. 本次代码补充

修改文件：

- `apps/backend/app/services/classifier.py`
- `apps/backend/app/services/solver.py`

本次补充内容：

1. 在 `classifier.py` 中显式捕获 `pydantic.ValidationError`；
2. 当 classifier 返回不兼容 schema 时，改为抛出：

```text
code=5005
classifier returned incompatible schema: ...
```

3. 在 `solver.py` 中显式捕获 `pydantic.ValidationError`；
4. 当 solve 最终 schema 仍不兼容时，改为抛出：

```text
code=5006
solve returned incompatible schema: ...
```

说明：

- 当前 `solver.py` 已有 `_normalize_solve_payload()`，因此很多原本会失败的 solve 字段缺失场景现在会被兼容吸收；
- 所以本轮更适合把“不兼容 schema”的显式错误主要落在 classifier 侧验证。

## 3. 远端实际回归方式

本轮在 `root@146.190.84.189` 上，使用 `note-backend` 容器镜像执行隔离测试，而不是依赖宿主机 Python 环境。

原因：

1. 远端宿主机 `python3` 未必安装 backend 全部依赖；
2. 容器镜像才是当前真实运行环境；
3. 这样得到的失败结果更接近真实部署行为。

## 4. 已验证失败路径与结果

### 4.1 缺少 `OPENAI_API_KEY`

结果：

```text
missing_key:code=5001;message=OPENAI_API_KEY is not configured
```

结论：

- 能明确定位为配置缺失，不是模糊 500。

### 4.2 错误 `OPENAI_BASE_URL` / 超时

结果：

```text
bad_base_or_timeout:code=5002;message=openai request failed: Request timed out.
```

结论：

- 能明确定位为上游请求失败 / 超时。

### 4.3 错误模型名

结果：

```text
bad_model:code=5002;message=openai request failed: Error code: 502 - {'error': {'message': 'unknown provider for model definitely-not-a-real-model', ...}}
```

结论：

- 能明确定位为模型名或 provider 错误。

### 4.4 classifier 返回不兼容 schema

通过 mock `OpenAIClient.chat_json()` 返回错误 JSON，结果：

```text
classifier_bad_schema:code=5005;message=classifier returned incompatible schema: 7 validation errors for ClassificationResult
```

结论：

- 这类问题现在会给出明确错误码与 schema 错误内容；
- 不再退化成泛化 internal error。

## 5. 本轮继续补齐：non-JSON 返回场景证据

本轮新增脚本：

- `scripts/ai_failure_check.py`

本轮继续补充代码：

- `apps/backend/app/services/classifier.py`
- `apps/backend/app/services/solver.py`

新增结果：

1. `classifier.py` 对原始 `json.loads()` 失败显式返回：

```text
code=5007
classifier returned invalid json: ...
```

2. `solver.py` 对原始 `json.loads()` 失败显式返回：

```text
code=5008
solve returned invalid json: ...
```

远端容器内实际验证输出：

```text
solver_non_json:code=5008;message=solve returned invalid json: Expecting value: line 1 column 1 (char 0)
classifier_non_json:code=5007;message=classifier returned invalid json: Expecting value: line 1 column 1 (char 0)
```

结论：

- `non-JSON` 返回场景现已形成独立证据；
- 关键 AI 失败路径已基本补齐。
