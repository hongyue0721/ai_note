# 阶段 2：鉴权实现记录

## 1. 本次目标

把“普通用户体系 + 独立管理员体系”从设计推进到真实接口层。

## 2. 本次已完成

### 2.1 新增鉴权请求/响应 schema

已新增：

- 用户注册请求
- 用户登录请求
- 管理员登录请求
- 用户 profile 返回结构
- 管理员 profile 返回结构
- 用户登录返回结构
- 管理员登录返回结构

### 2.2 新增鉴权接口

已新增路由：

1. `POST /v1/auth/register`
2. `POST /v1/auth/login`
3. `GET /v1/me`
4. `POST /v1/admin/auth/login`
5. `GET /v1/admin/me`

### 2.3 独立管理员体系落实

当前实现已满足：

1. 普通用户走 `users` 表；
2. 管理员走 `admin_users` 表；
3. 普通用户 token scope 为 `user`；
4. 管理员 token scope 为 `admin`；
5. 普通用户 token 与管理员 token 使用不同 secret。

### 2.4 远端导入级验证

已在远端 `/root/note/apps/backend` 完成：

- 依赖安装；
- `import app.main` 校验；
- 新增鉴权路由后的再次导入校验。

结果：

- `backend-import-ok`
- `backend-import-ok-auth`

说明当前后端结构在远端 Linux Python 3.12 环境下可成功加载。
