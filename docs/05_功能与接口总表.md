# 功能、接口与技术栈总表

本文档基于当前仓库代码整理，目标是用一份文档说明系统功能、全部后端接口用途，以及当前技术栈。

## 1. 系统功能总览

StarGraph AI（星图智学）是一个面向学习笔记管理、题目解析与后台运维管理的全栈系统。当前系统由 student frontend、admin frontend、FastAPI backend、parse-job worker 和 PostgreSQL 组成。

### 1.1 student 端功能

student 端面向普通用户，主要能力包括：

- 通过 `space_key` 进入专属空间并获取登录态
- 上传图片、PDF 或文本内容，完成文件入库
- 在正式入库前调用预览分类接口，获取标题、摘要、标签、学科等建议
- 创建与管理笔记
- 创建与管理题目解析相关内容
- 按关键词搜索笔记与题目
- 查看知识点概览与弱项标签
- 发起 AI 解题请求，并将结果以 Markdown 形式展示
- 将解题结果加入笔记

### 1.2 admin 端功能

admin 端面向管理员，当前分为两页：

- 数据管理：查看系统状态、接口指标、解析任务情况、用户笔记数量详情
- 管理设置：修改管理员用户名/密码，修改运行时文本模型与视觉模型配置

### 1.3 backend 功能

backend 提供统一 HTTP API，负责：

- 用户与管理员鉴权
- 文件上传策略生成、本地写盘与上传确认
- 内容预览分类
- 笔记与题目入库、查询、编辑、删除
- parse jobs 查询与重试
- review tasks 查询与审核决策
- 搜索、图谱、仪表盘与 AI 解题
- 管理员运行时模型配置读写

### 1.4 worker 功能

worker 独立轮询待处理 parse jobs，执行分类/解析流程，并回写任务结果与实体状态。它复用 backend 服务层逻辑，是异步解析链路的执行器。

## 2. 后端接口说明

## 2.1 通用返回格式

所有接口统一返回以下结构：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

当接口失败时，`code` 为非 0，`message` 为错误说明。

## 2.2 鉴权要求

- 无鉴权：健康检查、元信息、用户注册/登录/空间进入、管理员登录
- 用户鉴权：用户侧内容接口，使用用户 JWT
- 管理员鉴权：管理员侧接口、审核接口、管理端 parse-job 重试接口，使用管理员 JWT

## 3. 关键业务流程说明

### 3.1 上传并入库笔记

1. 调用 `/v1/files/upload-policy` 创建上传策略与文件记录。
2. 调用 `/v1/files/upload-local` 将文件写入本地目录。
3. 调用 `/v1/files/confirm` 确认文件元数据。
4. 调用 `/v1/preview/upload-tags` 获取标题、摘要、标签等预览信息。
5. 调用 `/v1/notes/confirm` 或 `/v1/ingestions` 完成入库。

### 3.2 AI 解题

1. 用户输入题目文本，必要时附带文件内容。
2. frontend 调用 `/v1/solve`。
3. backend 调用 OpenAI 兼容网关完成求解。
4. frontend 对结果进行 Markdown 渲染，并支持“加入笔记”。

### 3.3 parse-job 异步处理

1. backend 创建 parse job。
2. worker 轮询读取 pending job。
3. worker 执行解析/分类逻辑。
4. 成功后回写 `result_json` 与实体 `parse_status`。
5. 管理员可通过后台查看任务并发起重试。

## 4. 技术栈说明

## 4.1 后端技术栈

- **Python 3.11**：backend 与 worker 运行时
- **FastAPI 0.116.1**：HTTP API 框架
- **Uvicorn 0.35.0**：ASGI 服务运行器
- **Pydantic 2.11.7 / pydantic-settings 2.10.1**：配置管理与数据校验
- **SQLAlchemy 2.0.43**：ORM 与数据库访问
- **Alembic 1.16.5**：数据库迁移工具
- **psycopg 3.2.9**：PostgreSQL 驱动
- **python-jose 3.5.0**：JWT 生成与校验
- **passlib 1.7.4**：密码哈希与验证
- **python-multipart 0.0.20**：文件上传表单处理
- **httpx 0.28.1**：HTTP 客户端
- **openai 1.68.2**：OpenAI 兼容 AI 网关调用
- **pypdf 5.4.0**：PDF 文本抽取

## 4.2 前端技术栈

### student frontend

- **Vue 3.5.30**：前端框架
- **TypeScript 5.9**：开发语言
- **Vite 8**：开发与构建工具
- **marked 17.0.5**：Markdown 渲染
- **DOMPurify 3.3.3**：Markdown HTML 安全清洗

### admin frontend

- **Vue 3.5.30**：前端框架
- **TypeScript 5.9**：开发语言
- **Vite 8**：开发与构建工具

## 4.3 存储与部署技术栈

- **PostgreSQL 16**：主数据库
- **Docker Compose**：本地与线上编排
- **python:3.11-slim**：backend / worker 基础镜像
- **node:20-alpine**：前端构建镜像
- **nginx:stable-alpine**：前端静态资源容器
- **本地文件存储**：上传文件写入 `data/uploads`

## 4.4 当前架构特征

- student 与 admin 为两个独立前端应用，分别构建、分别部署
- backend 与 worker 共享 backend 服务层逻辑
- parse-job 采用异步处理，用户入库与后台解析分离
- runtime config 持久化，管理员可在线调整文本模型与视觉模型配置

## 5. 当前数据实体

当前系统围绕以下核心实体运行：

- `User`：student 用户
- `AdminUser`：管理员用户
- `FileRecord`：上传文件记录
- `Note`：笔记实体
- `Problem`：题目实体
- `ParseJob`：异步解析任务
- `ReviewTask`：审核任务
- `RuntimeConfig`：运行时模型配置
- `CanonicalTag`：标准标签

## 6. 结论

当前 StarGraph AI 已具备学习笔记管理、题目解答、上传预览分类、异步解析、搜索与知识点视图、管理员监控与模型配置等完整最小可用能力。接口边界已集中在 FastAPI backend 中，student/admin 前端均围绕这些接口完成业务交互，整体技术栈以 Python + FastAPI + PostgreSQL + Vue 3 + Docker Compose 为核心。
