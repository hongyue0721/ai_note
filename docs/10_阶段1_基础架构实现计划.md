# 阶段 1：基础架构实现计划

## 1. 阶段目标

把当前 `apps/backend` 从“只有最小 FastAPI 壳”的状态，推进到一个可以继续稳定开发业务功能的后端底座。

## 2. 当前现状

已确认：

1. 后端仅有 `/healthz` 和 `/v1/meta/app` 两个接口；
2. `requirements.txt` 仅包含 FastAPI / Pydantic 基础依赖；
3. 尚无数据库层、模型层、鉴权层、统一响应结构、日志结构；
4. worker 尚未接入任何任务机制；
5. 前端尚未实际初始化。

## 3. 阶段 1 本次准备实现的内容

### 3.1 后端依赖补全

计划加入：

- SQLAlchemy
- psycopg
- Alembic
- python-jose 或等价 JWT 库
- passlib / bcrypt 等密码哈希库
- httpx（后续 AI/服务调用可复用）
- python-multipart（上传预备）

### 3.2 目录结构补全

计划补齐：

- `app/db/`
- `app/models/`
- `app/services/`
- `app/api/deps/`
- `app/api/routes/v1/` 或等价分层结构
- `app/core/security.py`
- `app/core/logging.py`
- `app/core/exceptions.py`
- `app/schemas/common.py`

### 3.3 基础能力

计划建立：

1. 数据库引擎与会话
2. ORM Base
3. 时间戳基类
4. UUID 主键基类
5. 统一 API 返回结构
6. 基础异常处理
7. CORS 配置
8. 基础日志格式

### 3.4 首批数据模型（先打底）

先落“后续所有功能都会依赖”的基础模型：

1. 普通用户表
2. 管理员用户表
3. 文件表
4. parse_jobs 表
5. review_tasks 表

说明：
阶段 1 以打底为主，notes / problems / tags 等可在下一阶段继续补全。

## 4. 本阶段不做什么

1. 暂不实现完整业务 CRUD；
2. 暂不实现学生端前端；
3. 暂不实现后台前端；
4. 暂不实现完整 worker 流程；
5. 暂不实现完整测试集。

## 5. 验收标准

完成本阶段后，至少满足：

1. 后端目录结构清晰；
2. 可加载数据库配置；
3. 有统一响应结构；
4. 有基础模型层；
5. 有管理员与普通用户拆分基础；
6. LSP 诊断在修改文件上无新增语法错误。

## 6. 下一步

下一步开始实际修改后端代码与依赖，并同步记录实现结果。
