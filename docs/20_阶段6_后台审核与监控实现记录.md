# 阶段 6：后台审核与监控实现记录

## 1. 本次目标

把后台最关键的两块能力先落下来：

1. 审核任务接口
2. 基础监控/概览接口

## 2. 本次已完成

### 2.1 新增审核相关 schema 与接口

已新增：

- 审核任务列表返回结构
- 审核决策请求结构
- 审核决策返回结构

已新增接口：

1. `GET /v1/review/tasks`
2. `POST /v1/review/tasks/{task_id}/decision`

### 2.2 新增监控/概览接口

已新增：

1. `GET /v1/dashboard`
2. `GET /v1/admin/monitor/overview`

### 2.3 当前实现能力

管理员现在可以：

- 查询审核任务列表；
- 提交 approve / reject / replace 决策；
- 查看 parse job 总量、pending 数、failed 数；
- 查看待审核任务数；
- 查看最近错误信息列表。

## 3. 新发现的问题

在远端构造 review task 测试数据时发现：

- `review_tasks.user_id` 外键约束有效；
- 不能插入不存在的用户 UUID；

这不是 bug，而是数据库约束正常工作。

## 4. 额外修正

已把审核决策接口中的 `task_id` 参数从宽松字符串改为 `UUID`，降低后续后台调用歧义。

## 5. 下一步

下一步继续用**真实用户数据**构造审核任务与失败任务，完成后台审核/监控接口的远端验证。
