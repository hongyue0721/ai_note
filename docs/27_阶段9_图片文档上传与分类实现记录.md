# 阶段 9：图片 / 文档上传与分类实现记录

## 1. 本次目标

把当前上传链路从“只适合图片/错题演示”升级为：

1. 支持图片上传；
2. 支持文档上传；
3. 上传时明确分类；
4. 分类信息一路保留到 parse job 与结果中。

## 2. 本次已完成

### 2.1 后端数据结构扩展

已补充：

- `files.file_kind`
- `files.content_category`
- `notes.category`
- `problems.category`

### 2.2 后端接口扩展

已扩展：

- 上传策略接口支持 `file_kind`
- 上传策略接口校验支持图片与文档 mime type
- 文件确认接口返回 `file_kind` 与 `content_category`
- ingestion 支持 `content_category`
- parse job 查询返回分类信息

### 2.3 worker 结果扩展

当前 worker mock 结果中已保留：

- `content_category`
- `file_kind`

### 2.4 学生端上传界面扩展

已新增上传选择：

- 文件种类：`image` / `document`
- 内容分类：`problem` / `note` / `document`

并在最近任务区域展示：

- `entity_type`
- `content_category`
- `status`

## 3. 当前支持范围

当前优先支持的 mime type：

- `image/png`
- `image/jpeg`
- `application/pdf`
- `text/plain`
- `application/msword`
- `application/vnd.openxmlformats-officedocument.wordprocessingml.document`

## 4. 当前结论

系统现在已经具备“上传图片笔记与文档，并在上传阶段进行分类”的基础实现能力。

## 5. 下一步

下一步在远端执行新的真实验证，确认：

1. 图片分类上传可用；
2. 文档分类上传可用；
3. parse job 结果中保留分类字段。

## 6. 远端真实验证结果

已在远端完成“文档资料上传分类”真实验证，链路如下：

1. 用户登录；
2. 申请文档上传策略；
3. 确认文档上传；
4. 创建 ingestion；
5. worker 执行 parse job；
6. 查询结果。

### 已确认结果

1. `upload-policy` 返回 `accepted_as = document:document`
2. 文件确认接口返回：
   - `file_kind = document`
   - `content_category = document`
3. ingestion 返回：
   - `content_category = document`
4. worker 处理完成后 `result_json` 中保留：
   - `file_kind = document`
   - `content_category = document`

## 7. 当前结论

当前系统已经具备：

- 上传图片并分类
- 上传文档并分类
- 分类信息进入 parse job
- worker 处理后保留分类结果

这意味着你要求的“在使用时能够上传图片笔记与文档，并进行分类”，当前已经实现了后端真实链路。
