# 部署编排本地文件存储收口记录

## 1. 本次目标

让 `docker-compose.yml` 与已经固定的“本地 PostgreSQL + 本地文件存储”方向保持一致。

## 2. 本次完成内容

已完成：

1. 从 compose 中移除 `minio` 服务；
2. 移除 backend / worker 对 `minio` 的 `depends_on`；
3. 为 backend / worker 增加 uploads 目录挂载：
   - `./data/uploads:/app/data/uploads`

## 3. 为什么这样做

原因：

1. 当前项目已不再以 MinIO 作为主路径；
2. 如果 compose 仍保留 `minio`，会让下一位 AI 或部署执行者误判当前真实架构；
3. backend / worker 共享本地 uploads 目录，才能与当前本地文件存储方案一致。

## 4. 当前效果

收口后，compose 方向与当前代码现实保持一致：

- 数据库存储：postgres
- 文件存储：本地目录 `data/uploads`
