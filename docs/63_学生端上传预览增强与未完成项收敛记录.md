# 学生端上传预览增强与未完成项收敛记录

## 1. 本次目标

在不改动当前单页架构的前提下，先补一个最小但真实可交付的学生端上传 UX 增强，并把最新未完成事项再收敛一轮。

## 2. 本次前端修改

修改文件：

- `apps/frontend/student/src/App.vue`
- `apps/frontend/student/src/style.css`

本次新增：

1. 已选文件信息摘要；
2. 图片文件预览；
3. 非图片文件占位预览卡片；
4. 预览 URL 的安全释放逻辑。

## 3. 为什么先做这项

因为当前 student 端最明显、最小且最容易验证的产品缺口，就是上传区仍只有原生文件输入框，没有任何“我刚刚选中了什么”的反馈。

相比直接大改路由，这一项：

1. 改动更小；
2. 风险更低；
3. 能立刻提升上传主流程体验；
4. 更适合作为继续成品化前的第一步。
