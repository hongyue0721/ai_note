# 阶段 8：双前端实现计划

## 1. 目标

开始落地两个真实可演示的 Web 前端：

1. 学生端前端
2. 管理员后台前端

## 2. 当前策略

虽然原文档更偏 Uni-app，但当前比赛级交付更看重：

- 快速稳定
- Web 演示效果
- 与现有后端真实联调

因此当前前端实现先采用：

- **Vue 3 + TypeScript + Vite**
- 学生端一个前端项目
- 后台端一个独立前端项目

后续若需要再转 Uni-app/H5 多端，可在此基础上继续演进。

## 3. 视觉方向

本次前端风格定为：

- 深色科技感
- 信息层级清楚
- 卡片式布局
- 比赛展示友好
- 不做花哨拖拽特效

目标是让人第一眼看到：

> 这是一个已经成型、不是半成品的 AI 学习系统。

## 4. 学生端首批页面

1. 登录页
2. Dashboard
3. 上传页
4. 解析状态 / 最近任务
5. 图谱 / 弱点页（先做占位可视化）

## 5. 后台端首批页面

1. 管理员登录页
2. 审核任务页
3. 监控总览页
4. 解析任务页

## 6. 验收标准

1. 两个前端都能启动；
2. 登录页可用；
3. 学生端能走上传/创建任务；
4. 后台端能看审核任务和监控；
5. 页面视觉达到“比赛级成品”标准。
