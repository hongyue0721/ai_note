## P0-2：实现真实分类 Prompt 与分类 Service

### 目标

让系统在 worker 中对上传内容进行真实 AI 分类，而不是继续使用 mock 结果。

### 必做文件

- 新增：`apps/backend/app/services/classifier.py`
- 新增：`apps/backend/app/prompts/classify_prompt.txt` 或 `.md`
- 更新：`apps/backend/app/services/parse_jobs.py`

### 分类输出要求

模型必须返回结构化 JSON，至少包含：

```json
{
  "entity_type": "problem | note",
  "content_category": "problem | note | document",
  "subject": "math",
  "title": "提取出的标题或概述",
  "normalized_text": "清洗后的文本",
  "knowledge_candidates": [
    {"name": "一元二次方程", "confidence": 0.91}
  ],
  "confidence": 0.84,
  "needs_review": false,
  "review_reason": null
}
```

### 必做内容

1. 设计分类 prompt；
2. 明确要求模型只返回 JSON；
3. service 负责：
   - 组装 prompt
   - 调用 llm client
   - 解析 JSON
   - 做 schema 校验
   - 补默认值
4. `parse_jobs.py` 中将 mock 结果替换为 classifier 输出；
5. 当 `confidence < 阈值` 或 `needs_review=true` 时自动创建 review task；
6. 把 `llm_model` 写入 parse_job；

### 验收

- 新建 parse job 后，worker 执行能得到真实 AI 分类结果；
- `result_json` 含真实模型输出；
- 低置信度内容自动进入 `review_tasks`。

### 留痕

- 新增 `docs/35_AI分类Agent计划.md`
- 新增 `docs/36_AI分类Agent实现记录.md`

## P0-3：实现 Solve Prompt 与 Solve Service

### 目标

实现“本地题库未命中时，AI 给参考解析”的真实服务层。

### 必做文件

- 新增：`apps/backend/app/services/solver.py`
- 新增：`apps/backend/app/prompts/solve_prompt.txt`
- 可选新增：`apps/backend/app/prompts/solve_verifier_prompt.txt`

### 输出要求

按 `06_本地题库未命中时_LLM自主解题规划.md` 的结构执行，至少返回：

```json
{
  "subject": "math",
  "question_type": "calculation",
  "final_answer": "x=2",
  "solution_steps": ["..."],
  "knowledge_points": ["..."],
  "confidence": 0.78,
  "warnings": ["以下为 AI 参考解析"]
}
```

### 必做内容

1. 先做单阶段 solver；
2. 若时间允许，再补 verifier；
3. 必须明确写入“AI 参考解析”风险提示；
4. 结果不能覆盖本地题库标准答案；

### 验收

- 给定一道题目文本，可返回结构化 AI 参考解析；
- 返回中含 `warnings`；
- 返回中含 `confidence`。
