# StarGraph AI 开发进度与交接记录

## 项目定位

- 项目名称：星图智学（StarGraph AI）
- 当前目标：比赛演示版 MVP
- 本阶段原则：先搭可演示骨架，后补业务实现；先保留清晰边界，再逐步填充能力

## 本轮工作目标

1. 把当前纯文档仓库转换成可继续编码的项目骨架
2. 按文档约定建立前端、后端、worker、共享库、部署目录
3. 提供最小可运行入口与说明文件，方便下一个 AI 直接接手
4. 所有关键动作留痕，避免后续重复探索

## 当前已知限制

1. 目前原仓库只有方案文档，没有现成代码基础
2. 本轮优先保证结构可接手，不追求首轮就完成登录、上传、解析、图谱全链路
3. 前端首轮不直接引入完整 Uni-app 工程，先保留明确入口与接手说明，避免未确认脚手架细节时引入大量噪音文件
4. 数据库迁移、对象存储、Gemini 接入、JWT、解析任务状态机仍待后续实现

## 当前阻塞 / 风险

1. 还未真正初始化前端脚手架，后续需要决定是否严格使用 Uni-app CLI 产物，还是先 Web 演示页起步
2. 还未加入 Python 依赖管理与容器镜像定义的完整安装细节
3. 计划审查子代理曾出现超时，因此本轮以文档为准直接推进
4. 当前执行环境中的 `python` 缺少 `pip`，导致本机暂时无法直接安装 `apps/backend/requirements.txt`，LSP 因缺少依赖而出现第三方库未解析提示
