# 真实文件写盘闭环实现记录

## 1. 本次目标

把此前只完成“本地文件存储语义”的上传链路，补成**真实文件写盘闭环**。

## 2. 本次完成内容

已新增后端本地上传接口：

- `POST /v1/files/upload-local`

该接口行为：

1. 校验用户身份；
2. 校验 `file_id` 与 `object_key` 是否匹配当前用户文件记录；
3. 将上传文件内容写入：
   - `UPLOADS_ROOT_DIR / object_key`
4. 更新 `files.size_bytes` 与 `files.sha256`；
5. 返回实际写入字节数。

同时学生端已改造为：

1. 先调用 `upload-policy`
2. 再调用 `upload-local` 写文件内容
3. 再调用 `confirm`
4. 再调用 `ingestions`

## 3. 为什么这样做

这是最小修改方案：

- 不推翻现有 `upload-policy -> confirm -> ingestions` 主流程；
- 只在中间补一个真实写盘动作；
- 便于后续继续保留 policy/confirm 的兼容语义。

## 4. 当前验收标准

本次应验证：

1. 上传真实文件后，磁盘上出现真实文件；
2. `confirm` 返回的 `/uploads/...` 可访问该文件；
3. `files.size_bytes` 与 `files.sha256` 被真实更新；
4. 学生端链路未被破坏。
