## 3. 新发现的问题

在首次启动时发现：

- `passlib + bcrypt` 在当前远端环境下触发兼容/初始化问题；
- 问题集中在默认管理员 seed 的密码哈希阶段；
- 该问题会影响启动稳定性，因此必须修正。

## 4. 修正决策

为保证比赛级成品优先稳定落地，当前将密码哈希策略从：

- `bcrypt`

调整为：

- `pbkdf2_sha256`

原因：

1. 更稳，纯 Python 依赖链更简单；
2. 可避免当前环境中的 bcrypt backend 兼容问题；
3. 对比赛级完整成品已足够；
4. 后续若需要再迁移到更严格方案，也可平滑升级。
