# 阶段 9：图片 / 文档上传与分类计划

## 1. 目标

把当前上传链路升级为：

1. 支持图片上传；
2. 支持文档上传；
3. 支持用户在上传时指定分类；
4. 支持系统保存分类信息并进入解析链路。

## 2. 本阶段最小交付

### 后端

1. 扩展文件记录结构，支持文件类别与业务分类；
2. 扩展上传策略接口，支持图片 / PDF / TXT / DOC / DOCX；
3. 扩展 ingestion，支持文档资料类录入；
4. 在 parse job 结果中保留分类信息。

### 前端

1. 学生端上传区支持选择文件种类；
2. 支持选择业务分类：错题 / 笔记 / 文档资料；
3. 支持图片和文档两种展示路径；
4. 上传后把分类结果清楚展示出来。

## 3. 当前策略

比赛级阶段采用：

- 用户先手动选择一级分类；
- 系统在解析结果里保留分类信息；
- 后续再做更智能的自动细分类。

## 4. 支持范围

当前优先支持：

- `image/png`
- `image/jpeg`
- `application/pdf`
- `text/plain`
- `application/msword`
- `application/vnd.openxmlformats-officedocument.wordprocessingml.document`

## 5. 验收标准

1. 上传接口接受图片与文档；
2. 文件记录中保留分类信息；
3. ingestion 后可查询到分类相关结果；
4. 学生端上传界面能完整展示这一流程。
