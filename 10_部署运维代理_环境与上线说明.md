# 部署运维代理：环境、启动、上线与交付说明

## 你的角色

你负责把项目从“代码”变成“可运行系统”。

首月你的目标很清楚：

- 本地一键启动尽量简单
- 服务器部署可复现
- 环境变量清晰
- 出故障时能看日志
- 不要过度设计

---

## 一、首月推荐部署策略

## 1. 本地开发

使用：

- Docker Compose
- PostgreSQL
- MinIO
- backend
- worker
- frontend

## 2. 生产演示环境

使用：

- 一台 Linux 云主机
- Docker Compose
- PostgreSQL
- backend
- worker
- frontend
- Nginx
- 外部对象存储（七牛云）

---

## 二、你不应首月引入的内容

为了稳，不建议首月引入：

- Kubernetes
- Helm
- 服务网格
- 消息队列集群
- 分布式日志系统
- Prometheus + Grafana 全家桶（可二期再上）

你首月只要做到：

- 能起
- 能用
- 能查日志
- 能备份

---

## 三、推荐环境变量分组

## 1. 应用基础

- `APP_ENV`
- `APP_HOST`
- `APP_PORT`
- `JWT_SECRET`
- `CORS_ORIGINS`

## 2. 数据库

- `POSTGRES_HOST`
- `POSTGRES_PORT`
- `POSTGRES_DB`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `DATABASE_URL`

## 3. 存储

- `STORAGE_PROVIDER`
- `MINIO_ENDPOINT`
- `MINIO_ACCESS_KEY`
- `MINIO_SECRET_KEY`
- `MINIO_BUCKET`
- `QINIU_ACCESS_KEY`
- `QINIU_SECRET_KEY`
- `QINIU_BUCKET`
- `QINIU_DOMAIN`

## 4. AI

- `GEMINI_API_KEY`
- `GEMINI_MODEL_EXTRACT`
- `GEMINI_MODEL_SOLVE`

---

## 四、本地启动要求

你必须交付一份让非技术用户也能照着做的说明，至少包含：

1. 安装 Docker 与 Docker Compose
2. 执行启动命令
3. 如何访问前端
4. 如何访问后端接口文档
5. 如何查看日志
6. 如何停止服务

---

## 五、你必须提供的命令示例

至少包括：

```bash
docker compose up -d
docker compose ps
docker compose logs -f backend
docker compose logs -f worker
docker compose down
```

---

## 六、推荐上线步骤

## 步骤 1：准备服务器

- 安装 Docker
- 安装 Docker Compose
- 创建项目目录
- 上传 `.env`

## 步骤 2：准备反向代理

- Nginx 配置前端静态资源与 API 转发
- 确保 `/api` 转到 backend

## 步骤 3：数据库初始化

- 执行迁移或初始化脚本
- 建立管理员账号（如果需要）

## 步骤 4：启动服务

- 启动 PostgreSQL、backend、worker、frontend
- 验证健康检查接口

## 步骤 5：联调对象存储与 AI Key

- 上传测试图片
- 观察解析是否成功
- 确认 worker 可调用 Gemini

---

## 七、日志与排障最低要求

你必须保证以下日志是可看的：

- backend 日志
- worker 日志
- Nginx 日志
- PostgreSQL 容器状态

建议规则：

- 所有应用日志输出到 stdout/stderr
- 错误日志要包含 request id / job id / entity id
- 避免只返回“内部错误”而没有定位信息

---

## 八、备份最低要求

首月只做最小备份：

1. PostgreSQL 每日导出
2. `.env` 安全保存
3. 对象存储由云厂商负责持久化
4. 演示前额外导出一份数据库快照

---

## 九、你的验收标准

### 验收 1：本地可运行

- 拉起容器后可访问前端
- 后端接口文档可打开
- 数据库与 MinIO 正常

### 验收 2：生产可部署

- 服务器能按文档部署
- 上传、解析、查看流程可演示

### 验收 3：出问题能定位

- 能通过日志找到 backend 或 worker 失败原因
- 能判断是对象存储问题、数据库问题还是 AI 接口问题

---

## 十、对你最重要的一句要求

> 你的工作不是做一套完美运维体系，而是让一个不懂开发的人，也能按文档把项目启动起来并演示成功。
