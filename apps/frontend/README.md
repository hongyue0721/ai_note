# Frontend

前端目标：比赛演示优先，先做 **Web/H5 可演示体验**。

## 当前策略

本轮先不直接生成完整 Uni-app 脚手架，原因：

1. 当前仓库从纯文档起步
2. 首轮需要先把整体工程边界、接口契约、目录结构稳定下来
3. 直接引入完整脚手架会产生大量样板噪音，不利于后续 AI 快速聚焦 MVP 页面

## 计划中的首批页面

- 登录页
- Dashboard
- 上传页
- 解析状态页
- 错题/笔记详情页
- 标签审核页
- 图谱页（Web）
